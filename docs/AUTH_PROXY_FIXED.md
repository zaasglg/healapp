# ✅ Исправлено проксирование Auth

## Проблема

Auth возвращал 404, потому что nginx неправильно проксировал запросы.

## Решение

**Было:** `proxy_pass http://127.0.0.1:54326/auth/;` (добавлял лишний /auth/)
**Стало:** `proxy_pass http://127.0.0.1:54326/;` (правильное проксирование)

Когда nginx получает запрос `/auth/v1/token`, он проксирует его на `http://127.0.0.1:54326/auth/v1/token`, что правильно, так как Auth слушает на корневом пути `/auth/v1/`.
