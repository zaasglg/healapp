# ✅ Исправлено проксирование Auth с rewrite

## Проблема

Auth слушает на корневом пути `/`, а не на `/auth/v1/`, поэтому запросы `/auth/v1/token` возвращали 404.

## Решение

Добавлен `rewrite` для удаления `/auth` из пути перед проксированием:

```nginx
rewrite ^/auth/(.*) /$1 break;
proxy_pass http://127.0.0.1:54326;
```

Теперь:
- Запрос `/auth/v1/token` → проксируется на `http://127.0.0.1:54326/v1/token`
- Запрос `/auth/v1/settings` → проксируется на `http://127.0.0.1:54326/v1/settings`
