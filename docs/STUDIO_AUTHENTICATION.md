# Аутентификация для Supabase Studio

## Настройка базовой HTTP аутентификации

Для защиты Supabase Studio паролем (как в видео) настроена базовая HTTP аутентификация.

### Учетные данные для доступа:

- **Логин**: `admin`
- **Пароль**: `HealApp2024SecurePass!`

### Как изменить пароль:

```bash
# На сервере
htpasswd /etc/nginx/.htpasswd admin
# Введите новый пароль при запросе
```

### Как добавить нового пользователя:

```bash
# На сервере
htpasswd /etc/nginx/.htpasswd username
# Введите пароль при запросе
```

### Как удалить пользователя:

```bash
# На сервере
htpasswd -D /etc/nginx/.htpasswd username
```

## Доступ к Studio

После настройки файрвола и применения правил:

1. Откройте `http://supabase.healapp.ru/` в браузере
2. Появится окно запроса логина и пароля
3. Введите:
   - **Логин**: `admin`
   - **Пароль**: `HealApp2024SecurePass!`
4. Откроется интерфейс Supabase Studio

## API эндпоинты

API эндпоинты (`/rest/`, `/functions/`, `/auth/`) работают **без аутентификации** - они нужны для работы вашего приложения.

Только Studio защищен паролем.
