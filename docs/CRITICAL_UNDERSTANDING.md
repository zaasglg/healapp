# КРИТИЧЕСКОЕ ПОНИМАНИЕ ПРОБЛЕМЫ

## Главная проблема

**Пароль для `postgres` устанавливается ТОЛЬКО при первой инициализации БД.**

Когда вы пересоздаете контейнер БД, но volume уже существует - пароль НЕ обновляется!

## Решение

### Удалить volume и пересоздать БД

**ВНИМАНИЕ:** Это удалит все данные в БД!

```bash
cd ~/HealApp-Web

# Остановить БД
docker compose stop db

# Удалить volume (удалит данные!)
docker volume rm healapp-web_postgres-data

# Запустить БД заново (пароль установится из .env)
docker compose up -d db

# Подождать 40 секунд
sleep 40

# Перезапустить meta
docker compose restart meta

# Перезапустить Studio
docker restart supabase-studio
```

## Важно про Studio

**Self-hosted Supabase Studio НЕ имеет системы аккаунтов как в облачной версии.**

- **Облачная версия:** Вы входите в свой аккаунт → видите свои проекты → выбираете проект → работаете с ним
- **Self-hosted версия:** Studio просто работает с одной БД напрямую, нет системы аккаунтов и проектов

Studio в self-hosted версии - это просто веб-интерфейс для управления БД, а не полноценная платформа.

## Что вы увидите после исправления

1. Откроете `http://176.124.217.224/`
2. Введете HTTP аутентификацию: `admin` / `HealApp2024SecurePass!`
3. Увидите интерфейс Studio с вашими таблицами
4. **НО:** Это будет просто интерфейс для управления БД, без системы аккаунтов

## Если нужна система аккаунтов

Self-hosted Supabase не поддерживает систему аккаунтов из коробки. Это просто API сервер и веб-интерфейс для управления БД.

Если в видео показана система аккаунтов - возможно, там используется:
- Кастомная настройка
- Или другая версия/форк Supabase
- Или это просто демонстрация облачной версии
