# Настройка файрвола в Timeweb Cloud

## Проблема

Домен `supabase.healapp.ru` не доступен с вашего компьютера, хотя с сервера он работает.

## Решение

### 1. Откройте порт 80 в панели управления Timeweb Cloud

1. Войдите в панель управления Timeweb Cloud
2. Найдите ваш сервер (IP: 176.124.217.224)
3. Перейдите в раздел "Безопасность" или "Firewall"
4. Создайте группу правил Firewall (если еще не создана)
5. В разделе "Входящий" (Incoming) нажмите "+ Добавить"
6. В модальном окне "Добавить правило" настройте:
   - **Область действия**: "Для всех адресов" (For all addresses) ✅
   - **Тип**: "Своё правило" (Custom rule) ✅
   - **Протокол**: "TCP" ✅
   - **Порт или диапазон портов**: `80` ✅
7. Нажмите "Добавить" (Add)
8. Примените группу правил к вашему серверу

### 2. Проверьте группу безопасности (Security Groups)

Если используется группа безопасности:
1. Найдите группу безопасности для вашего сервера
2. Добавьте правило для порта 80 (TCP, входящий трафик)

### 3. Альтернатива: Используйте другой порт

Если порт 80 заблокирован провайдером, можно настроить nginx на другой порт (например, 8080) и открыть его:

```bash
# На сервере
# Измените /etc/nginx/sites-available/supabase:
# listen 8080; вместо listen 80;
```

Но лучше открыть порт 80 в панели управления.

## Проверка после настройки

После открытия порта 80:
1. Подождите 1-2 минуты
2. Откройте `http://supabase.healapp.ru/` в браузере
3. Должен открыться Supabase Studio

## Если порт 80 недоступен

Используйте прямой доступ через IP и порт:
- `http://176.124.217.224:54324/` - Studio напрямую
