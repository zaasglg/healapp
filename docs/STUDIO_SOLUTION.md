# Решение проблемы Supabase Studio

## Текущая ситуация

`postgres-meta` настроен правильно:
- `PG_META_DB_USER=postgres`
- `PG_META_DB_PASSWORD=Dn2907200!`

Но Studio все еще показывает ошибки. Это может быть из-за кэша браузера или старых соединений.

## Решение

### 1. Полностью очистить кэш браузера

1. Откройте `http://supabase.healapp.ru/` в браузере
2. Нажмите `Ctrl+Shift+Delete` (Windows) или `Cmd+Shift+Delete` (Mac)
3. Выберите "Кэшированные изображения и файлы"
4. Нажмите "Очистить данные"
5. Закройте и откройте браузер заново

### 2. Использовать режим инкогнито

1. Откройте браузер в режиме инкогнито (`Ctrl+Shift+N` или `Cmd+Shift+N`)
2. Перейдите на `http://supabase.healapp.ru/`
3. Введите HTTP аутентификацию: `admin` / `HealApp2024SecurePass!`

### 3. Проверить работу напрямую

Если проблема сохраняется, проверьте работу `meta` API:

```bash
# На сервере
curl http://127.0.0.1:8080/tables?included_schemas=public
```

Если API возвращает данные, значит проблема в Studio или браузере.

## Альтернативное решение

Если Studio все еще не работает, можно использовать прямой доступ к БД через `psql` или другие инструменты для управления данными.
