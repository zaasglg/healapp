# ✅ Исправление пользователя для meta сервиса

## Проблема

Supabase Studio не может подключиться к БД, потому что:
- Meta сервис пытается подключиться как `supabase_admin`
- Но в конфигурации был указан `PG_META_DB_USER: postgres`
- Согласно документации Supabase, meta должен использовать `supabase_admin`

## Решение

1. **Обновлена конфигурация docker-compose.yml:**
   - Изменено `PG_META_DB_USER: postgres` на `PG_META_DB_USER: supabase_admin`

2. **Пароль для supabase_admin:**
   - Пароль установлен: `Dn2907200!` (такой же как у postgres)
   - Пользователь `supabase_admin` уже существует в БД

## Статус

- ✅ Конфигурация meta обновлена
- ✅ Пользователь `supabase_admin` использует правильный пароль
- ✅ Meta сервис перезапущен
