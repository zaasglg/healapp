# ВАЖНО: Self-hosted Supabase Studio

## Ключевое отличие от облачной версии

**Self-hosted Supabase Studio НЕ имеет системы аккаунтов как в облачной версии.**

### Облачная версия Supabase:
- Вы входите в свой аккаунт (email/пароль)
- Видите список своих проектов
- Выбираете проект
- Работаете с проектом

### Self-hosted версия Supabase:
- **Нет системы аккаунтов**
- **Нет списка проектов**
- Studio просто работает с **одной БД напрямую**
- Это просто веб-интерфейс для управления БД

## Что вы увидите в Studio

После исправления проблемы с паролем:

1. Откроете `http://176.124.217.224/`
2. Введете HTTP аутентификацию: `admin` / `HealApp2024SecurePass!`
3. Увидите интерфейс Studio
4. Сразу увидите все таблицы вашей БД
5. **НО:** Это будет просто интерфейс для управления БД, без системы аккаунтов

## Если в видео показана система аккаунтов

Возможно:
- Это демонстрация облачной версии
- Или используется кастомная настройка
- Или другая версия/форк Supabase

## Что можно сделать

Self-hosted Supabase - это просто API сервер и веб-интерфейс для управления БД. Если нужна система аккаунтов - нужно разрабатывать свою или использовать облачную версию.
