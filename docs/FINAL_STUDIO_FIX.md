## Что нужно сделать

### 1. Очистить кэш браузера

**Важно:** Браузер может кэшировать старые ошибки!

1. Откройте `http://supabase.healapp.ru/` в браузере
2. Нажмите `F12` для открытия DevTools
3. Правой кнопкой на кнопку обновления страницы
4. Выберите "Очистить кэш и жесткая перезагрузка" (Empty Cache and Hard Reload)
5. Или используйте `Ctrl+Shift+R` (Windows) / `Cmd+Shift+R` (Mac)

### 2. Использовать режим инкогнито

1. Откройте браузер в режиме инкогнито (`Ctrl+Shift+N`)
2. Перейдите на `http://supabase.healapp.ru/`
3. Введите HTTP аутентификацию: `admin` / `HealApp2024SecurePass!`

### 3. Проверить работу

После очистки кэша вы должны увидеть:
- ✅ Все таблицы в разделе "Table Editor"
- ✅ Возможность открыть любую таблицу
- ✅ Возможность редактировать данные
- ✅ Доступ ко всем функциям Studio

## Если проблема сохраняется

Проверьте работу `meta` API напрямую:

```bash
# На сервере
curl http://127.0.0.1:8080/tables?included_schemas=public
```

Если API возвращает данные, значит проблема в браузере/кэше, а не в сервере.

## Важно

Self-hosted Supabase Studio работает с одним проектом напрямую. Нет системы аккаунтов как в облачной версии. Вы просто входите через HTTP аутентификацию и получаете доступ ко всем данным БД.
