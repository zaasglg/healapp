**Пароль для `postgres` устанавливается ТОЛЬКО при первой инициализации БД.**

Когда volume уже существует, пароль НЕ обновляется при пересоздании контейнера.

## Что я сделал

1. ✅ Пересоздал БД с удалением volume (пароль теперь правильный: `Dn2907200!`)
2. ✅ Перезапустил `meta` сервис
3. ✅ Перезапустил Studio

**ВНИМАНИЕ:** Данные в БД могли быть потеряны при удалении volume. Если у вас был бэкап - нужно восстановить.

## Важно про Studio

### Self-hosted Supabase Studio НЕ имеет системы аккаунтов!

**Облачная версия:**
- Вход в аккаунт (email/пароль)
- Список проектов
- Выбор проекта
- Работа с проектом

**Self-hosted версия:**
- **НЕТ системы аккаунтов**
- **НЕТ списка проектов**
- Studio работает с **одной БД напрямую**
- Это просто веб-интерфейс для управления БД

### Что вы увидите

1. Откройте `http://176.124.217.224/` в режиме инкогнито
2. Введите HTTP аутентификацию: `admin` / `HealApp2024SecurePass!`
3. Увидите интерфейс Studio
4. Сразу увидите все таблицы вашей БД (если данные восстановлены)
5. **НО:** Это будет просто интерфейс для управления БД, без системы аккаунтов
