# АБСОЛЮТНОЕ ИСПРАВЛЕНИЕ

## Проблема

`postgres-meta` не может подключиться к БД, даже после установки пароля.

## Решение

### 1. Установить пароль через интерактивную команду

```bash
docker exec -it caregivers-diary-db psql -U postgres
# Затем в psql:
ALTER USER postgres WITH PASSWORD 'Dn2907200!';
\q
```

### 2. Перезапустить meta

```bash
cd ~/HealApp-Web
docker compose restart meta
```

### 3. Проверить подключение

```bash
curl http://127.0.0.1:8080/tables?included_schemas=public
```

## Если не работает

Возможно, нужно пересоздать БД с правильным паролем с самого начала, но это удалит данные. Лучше сначала попробовать установить пароль через интерактивную команду.

## Альтернативное решение

Если проблема сохраняется, можно временно изменить `pg_hba.conf` на `trust` для локальных подключений, но это небезопасно для production.
