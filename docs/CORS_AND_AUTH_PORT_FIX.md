# ✅ Исправлена ошибка CORS и порт Auth

## Проблемы

1. **CORS ошибка**: `Request header field x-supabase-api-version is not allowed`
2. **Нет ответа в Network**: Auth не отвечает на запросы

## Решение

### 1. Исправлен порт для Auth проксирования ✅

**Было:** `proxy_pass http://127.0.0.1:54323/auth/;` (неправильный порт)
**Стало:** `proxy_pass http://127.0.0.1:54326/auth/;` (правильный порт)

Auth слушает на порту **54326** (внешний порт), который маппится на внутренний порт 9999.

### 2. Добавлен заголовок x-supabase-api-version ✅

Добавлен в список разрешенных заголовков для всех эндпоинтов.

### 3. Обновлена конфигурация nginx ✅

- ✅ CORS заголовки для всех запросов
- ✅ Обработка OPTIONS запросов (preflight)
- ✅ Правильный порт для Auth
